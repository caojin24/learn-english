# 6岁少儿英语启蒙软件需求文档（按当前代码实现更新）

## 一、文档说明

### 1.1 文档目的

本文档用于基于当前仓库代码的真实实现状态，整理本项目的产品需求、功能范围、技术约束和后续迭代边界。

本次更新遵循一个原则：以现有代码为准，不再把未落地的设想与已实现能力混写。

### 1.2 产品定位

- 产品类型：面向低龄儿童的英语启蒙单页应用
- 目标用户：6 岁左右儿童及其家长
- 核心目标：通过“听、说、玩、看”的低压力互动，帮助孩子建立英语兴趣
- 使用方式：无需登录、无需后端、打开网页即可使用
- 部署形态：纯前端项目，学习设置和进度保存在浏览器本地

### 1.3 当前版本原则

- 优先简单直接，减少复杂操作
- 强调鼓励反馈，不做考试式打分
- 以移动端触屏操作为优先
- 先保证体验闭环，再考虑内容扩充

## 二、当前版本范围总览

### 2.1 当前已实现页面

- 首页
- 家长设置页
- 分级听力页
- 跟读练习页
- 游戏模块页
- 看图识词页
- 日常短句页
- 魔法口袋页
- 绘本动画页

### 2.2 当前首页实际入口

首页当前展示 6 个学习卡片入口：

- 分级听力
- 看图识词
- 日常短句
- 魔法口袋
- 绘本动画
- 游戏模块

家长设置通过首页右上角齿轮按钮进入。

跟读练习页面虽然已经实现，但当前没有在首页直接暴露入口，也没有其他页面跳转到该页。

### 2.3 当前内容规模

- 短句/听力内容：88 句
- 识词内容：180 个
- 游戏词库内容：603 个，来自 `src/data/word.json`
- 视频内容：15 条
- 单词分类：全部、动物、水果、颜色、数字、日常用品、动作
- 短句场景分类：推荐、打招呼、自我表达、晨间日常、吃饭时间、玩耍互动、情绪需求、睡前、简单问答
- 魔法口袋内容：可分别收集单词和短句，数量随学习过程变化

## 三、用户需求

### 3.1 孩子端需求

- 页面按钮大、反馈明确、操作步骤少
- 听力和短句内容可以重复播放
- 跟读不打分，只给鼓励反馈
- 完成练习后有星星、徽章、toast 等正向激励
- 内容贴近日常生活，适合低龄儿童理解和模仿
- 遇到暂时不会的单词或短句时，可以先收进专门的复习口袋，后面重复练习

### 3.2 家长端需求

- 能查看当前统一难度并调整发音偏好
- 能限制单次动画观看时长
- 刷新页面后设置和进度不丢失
- 不需要登录注册，不依赖后台系统

### 3.3 当前版本未覆盖的家长需求

以下内容在当前代码中尚未实现，不应视为现有能力：

- 每日学习时长统计
- 学习报告或复杂数据看板
- 学习模块开关管理
- 进度重置与二次确认流程
- 完整的内容后台或可视化配置系统

## 四、核心功能需求

### 4.1 首页

#### 功能描述

首页用于承接孩子的首次进入和模块选择，并展示当前奖励数量。

#### 当前实现

- 顶部显示产品标题“宝贝英语启蒙”和简短副标题
- 右上角提供设置按钮
- 中部显示 6 张学习模块卡片
- 首页卡片区下方展示奖励信息，包含星星和徽章数量
- 点击模块卡片后切换到对应页面

#### 需求要求

- 保持首页结构简单，不增加复杂说明文字
- 首页仍应优先服务孩子自主点击
- 若未来增加模块入口，需要评估是否会破坏当前的低认知负担

### 4.2 魔法口袋

#### 功能描述

`魔法口袋` 用来收集孩子暂时不会的内容，避免当下卡住，也方便后续集中复习。

#### 当前实现

- 首页提供独立入口
- 支持收集两类内容：
  - 单词
  - 短句
- 在看图识词页，可以把当前目标单词放进魔法口袋
- 在日常短句页，可以把当前短句放进魔法口袋
- 魔法口袋页支持在“单词口袋”和“短句口袋”之间切换
- 进入口袋后可逐条循环查看
- 单词支持再次播放发音
- 短句支持再次播放并逐词高亮
- 已掌握的内容可以从口袋中移除
- 口袋内容和当前页签会保存在本地

#### 需求要求

- 入口和文案要保持温和、鼓励式，避免出现“错题本”这类压力感更强的表达
- 复习流程应尽量短，不引入复杂筛选、排序或多级目录
- 后续若扩展到听力或跟读内容，优先复用现有口袋结构

### 4.3 家长设置

#### 当前实现项

- 分级听力难度：固定为基础
- 跟读练习难度：固定为基础
- 日常短句难度：固定为基础
- 看图识词分类选择
- 发音偏好：美式 / 英式
- 单次动画观看时长：5 / 10 / 15 分钟

#### 需求要求

- 设置项保持精简，只保留对学习体验有直接影响的配置
- 所有设置自动保存到本地
- 不引入账号体系和云端同步前，不要把设置设计为跨设备能力

### 4.4 分级听力

#### 功能描述

通过短句播放和逐词高亮，让孩子反复听英语表达。

#### 当前实现

- 内容来自统一短句数据源
- 当前统一使用 `basic` 难度，对应完整 88 句内容
- 每屏显示 1 句
- 显示 emoji 形式的句子场景图
- 顶部支持“全部 + 具体场景分类”切换，默认进入“全部”
- 在“全部”下按完整句库顺序浏览，并显示全局序号
- 在具体分类下按分类内部顺序浏览，并显示分类内序号
- 支持播放发音
- 支持逐词高亮
- 支持“上一句”“下一句”
- 具体分类的最后一句点击“下一句”时，会直接进入下一个分类的第一句
- 到达分类最后一句时，页面会提前提示“点下一句将进入下一个分类”
- 在当前浏览范围内，每正常前进 4 句后，会固定插入 1 句已听过内容进行回忆复习
- 完成听力会记录当前句子 ID
- 每听完第 5、10、15... 句时奖励 1 颗星星

#### 音频要求

- 优先调用 `/api/tts` 生成稳定发音
- 当前 `/api/tts` 由 Vercel Python Function 提供，底层直接调用 `edge-tts`
- 若接口失败，回退浏览器 `speechSynthesis`
- 切换“上一句”“下一句”或切换分类时，需要中断当前播放，并取消进行中的 TTS 请求
- 发音以慢速、清晰、适合儿童模仿为目标

#### 当前边界

- 没有真正的“连续播放列表”模式
- 没有单独的图片资源接入，当前以 emoji 为主

### 4.5 跟读练习

#### 功能描述

通过麦克风录音和回放降低孩子开口门槛。

#### 当前实现

- 内容与听力模块复用同一批句子
- 当前统一使用 `basic` 难度，对应完整句库
- 中央麦克风按钮控制录音开始和停止
- 自动在 5 秒后停止录音
- 录音成功后可直接回放
- 若录音内容过短或音量太小，会提示重试
- 若麦克风权限未授权，会展示友好提示
- 跟读成功后给予鼓励 toast，并奖励 1 颗星星

#### 当前边界

- 页面已实现，但当前没有首页入口
- 不做语音识别、评分、纠音或 AI 评测

### 4.6 看图识词

#### 功能描述

通过“点击识词”和“配对游戏”帮助孩子建立单词和图像的联想。

#### 当前实现

- 支持两种模式：
  - 点击识词
  - 配对游戏
- 每轮展示 3 个单词
- 当前目标图使用 emoji 展示
- 支持切换分类，并可在页面内直接打开分类选择器
- 支持播放单词发音
- 支持把当前目标单词放进魔法口袋或从口袋中移出
- 发音优先使用有道词典发音地址，失败时回退浏览器 TTS
- 连续完成 3 个单词奖励 1 颗星星
- 每累计完成 15 个单词时额外奖励 1 枚徽章
- 支持“换一组”继续练习

#### 需求要求

- 保持低门槛交互，不引入拖拽、复杂连线等高摩擦形式
- 单词数据应继续统一维护在 `src/data/content.ts`
- 若后续接入真实图片资源，应保留 emoji 或默认图作为兜底

### 4.7 游戏模块

#### 功能描述

通过“英文单词 + 中文短释义”配对消除的方式，提供一个独立的轻量小游戏入口。

#### 当前实现

- 首页提供独立入口，位于模块卡片列表中
- 词库直接来自 `src/data/word.json`
- 在保留原始 `meaning` 的同时，为每个词条新增 `gameMeaningZh` 作为游戏短释义
- 当前每局默认随机抽取 6 组词卡，生成英文卡和中文卡
- 交互顺序为：先点英文，再点中文；配对成功后两张卡一起消失
- 同一局会避免抽到重复英文和重复中文短释义，降低无解题概率
- 选错时会给出文字提示，并短暂高亮错误卡片
- 匹配成功时会显示成功提示，并只请求播放一个有道单词发音地址
- 当前不接入星星/徽章奖励，也不接入魔法口袋

#### 需求要求

- 保持点击式低门槛交互，不引入拖拽、连线或翻牌记忆玩法
- 游戏提示要短、直观、适合儿童快速理解
- 游戏词库扩展时，优先继续维护 `gameMeaningZh` 的唯一性和可区分性

### 4.8 日常短句

#### 功能描述

通过每日推荐和分类浏览，让孩子在生活场景里反复接触常用表达。

#### 当前实现

- 当前统一使用 `basic` 难度，对应完整 88 句内容
- 顶部支持分类切换
- 提供“推荐”分组
- 每次推荐最多 5 句
- 推荐句子会保存在本地进度中
- 支持“换今天推荐”手动刷新推荐
- 支持播放短句和逐词高亮
- 支持“上一句”“下一句”
- 支持把当前短句放进魔法口袋或从口袋中移出
- 播放完成后记录短句完成状态，并奖励 1 颗星星
- 句子展示包含英文和中文释义
- 切换“上一句”“下一句”或切换分类时，会停止当前播放，并取消进行中的 TTS 请求

#### 推荐逻辑要求

- 当前推荐优先从未完成句子中选择
- 推荐结果需要可持久化，避免刷新后完全丢失
- 如果未来要调整推荐算法，优先改 `src/lib/recommendations.ts`

#### 当前边界

- 当前没有“跳转到跟读练习”的按钮
- 当前使用 emoji，不是场景插画

### 4.9 绘本动画

#### 功能描述

通过英文绘本和启蒙视频提供轻量输入。

#### 当前实现

- 视频列表展示 emoji 封面、英文标题、中文标题和摘要
- 当前共 15 条视频数据
- 资源来源字段为 B 站
- 列表点击后进入详情播放态
- 详情页通过 B 站 `iframe` 播放器展示内容
- 同时提供“去 B 站播放”的外链按钮
- 根据设置中的观看上限计时
- 超过时长上限后停止当前嵌入播放，并提示休息

#### 当前边界

- 当前不是本地 MP4 播放
- 不支持原生进度条控制逻辑的自定义开发
- 视频稳定性受外部站点可用性影响

## 五、奖励与进度需求

### 5.1 奖励机制

当前奖励由全局 `RewardToast` 和进度状态共同驱动：

- 听力：每完成 5 句奖励 1 颗星星
- 跟读：每次有效录音奖励 1 颗星星
- 识词：每完成 3 个单词奖励 1 颗星星；每完成 15 个单词奖励 1 枚徽章
- 短句：每次播放完成奖励 1 颗星星
- 视频：当前只有鼓励 toast，不累计星星和徽章
- 游戏模块：当前不累计星星和徽章

### 5.2 进度持久化

当前保存在本地的进度包括：

- 家长设置
- 已听过句子 ID
- 已完成跟读句子 ID
- 已完成单词 ID
- 已完成短句 ID
- 魔法口袋单词 ID
- 魔法口袋短句 ID
- 奖励数量
- 推荐短句 ID
- 各模块最近浏览位置

当前不保存在本地的内容包括：

- 游戏模块当前局面和回合进度

### 5.3 持久化要求

- 继续使用 `localStorage`
- 保持数据结构向后兼容
- 新增字段时必须同步更新默认值和归一化逻辑

## 六、界面与交互需求

### 6.1 整体视觉方向

- 温暖、轻松、适合儿童
- 使用大圆角、柔和底色和明显点击反馈
- 保留当前轻量卡片式布局

### 6.2 交互要求

- 以点击为主，避免复杂手势
- 按钮最小尺寸应保持适合触屏
- 奖励反馈简短直接，不干扰主流程
- 设置页允许中文说明，其余学习流程避免长篇说明

### 6.3 当前展示素材策略

- 首页和学习页当前大量使用 emoji 作为内容主视觉
- 当前仓库已经移除图片资源目录
- 后续若重新接入真实图片，要避免一次性大改全部模块

## 七、技术需求

### 7.1 技术栈

- React 18
- TypeScript
- Tailwind CSS
- Vite
- Vercel Python Functions
- Python `edge-tts`
- 浏览器 `localStorage`
- 浏览器 `speechSynthesis`
- 浏览器 `MediaRecorder`

### 7.2 架构要求

- 保持单页应用模式
- 页面切换继续由 `src/App.tsx` 内部状态控制
- 不默认引入 `react-router`
- 静态内容继续统一放在 `src/data/content.ts`

### 7.3 音频能力要求

- 单词发音：优先有道发音地址，失败时回退浏览器 TTS
- 游戏模块匹配成功发音：当前只请求并播放一个有道单词发音地址
- 听力和短句发音：优先 `/api/tts`，当前由 Vercel Python Function 直接调用 `edge-tts`，失败时回退浏览器 TTS
- 跟读录音：依赖浏览器麦克风权限和 `MediaRecorder`

### 7.4 错误处理要求

- 本地存储失败时静默降级，不影响页面继续使用
- 麦克风权限失败时给出简洁提示
- 音频播放失败时允许回退到备用方案
- 视频嵌入异常时允许用户跳转外链继续播放

## 八、目录与实现映射

- `src/App.tsx`：页面切换、奖励逻辑、进度同步

- `src/pages/ListeningPage.tsx`：听力练习
- `src/pages/SpeakingPage.tsx`：跟读练习
- `src/pages/GamesPage.tsx`：游戏模块
- `src/pages/WordsPage.tsx`：识词玩法
- `src/pages/PhrasesPage.tsx`：短句练习和推荐
- `src/pages/PocketPage.tsx`：魔法口袋复习页
- `src/pages/VideosPage.tsx`：视频列表和播放态
- `src/pages/SettingsPage.tsx`：家长设置

- `src/data/word.json`：游戏模块原始词库与短释义

- `src/lib/recommendations.ts`：推荐逻辑
- `api/tts.py`：Vercel Python TTS 接口
- `requirements.txt`：Python 依赖

### 8.2 需求与代码同步原则

- 需求文档应优先反映当前代码行为
- 如果某项只是规划，不应描述成已实现功能
- 若后续恢复跟读入口、扩展魔法口袋范围、接入本地视频或重新引入图片，需要同步更新本文件

## 九、后续迭代建议

以下方向适合放入后续版本，而不是回写成当前能力：

- 将跟读练习重新接回首页或短句页
- 在魔法口袋里增加简单的掌握度标记或复习节奏
- 为视频页切换到本地可控的视频播放方案
- 为 `api/tts` 增加缓存、监控或更细的性能观测
- 增加进度重置、学习记录和基础测试

## 十、结论

当前项目已经具备一个可用的儿童英语启蒙前端闭环，重点是：

- 有明确的首页入口和设置入口
- 有听力、跟读、游戏、识词、短句、魔法口袋、视频七类核心学习能力
- 有本地持久化和奖励反馈
- 有清晰的后续扩展空间

但同时也要明确：

- 当前首页开放了 6 个学习入口
- 跟读页属于“已实现但未接入主入口”的状态
- 游戏模块已接入主入口，但当前仍是独立轻量玩法
- 视频模块依赖 B 站外部资源
- 当前展示层已经统一为 emoji 和文字

后续所有需求讨论，建议都基于以上现状继续推进。
